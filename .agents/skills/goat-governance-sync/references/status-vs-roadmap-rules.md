# Status vs Roadmap Rules

Apply these distinctions strictly:

- `ROADMAP.md`
  - unfinished work
  - follow-ons
  - open risks still inside the repository
- `PROJECT_STATUS.md`
  - shipped baseline
  - landed governance state
  - current known boundaries and external residual conditions

If the repo already has working code, tests, and workflows, that does not automatically mean the change belongs in both files. Put it where the document's role says it belongs.
